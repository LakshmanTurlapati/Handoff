<objective>
Integrate the local `codex app-server` stdio protocol into the bridge and normalize its thread/turn/approval traffic into the existing Codex Mobile live-session contracts.

Purpose: Phase 3 already built the browser/relay live shell on top of `packages/protocol/src/live-session.ts`. This plan backfills the real local bridge side so the relay stops depending on synthetic data and instead receives structured Codex events from the developer machine.

Output: Codex stdio adapter, event normalizer, and unit tests for listing, resume/history, and approval-preserving event translation.
</objective>
