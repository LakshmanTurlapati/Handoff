<objective>
Capture the active-session context and mint or reuse thread-bound handoff metadata at command invocation time.

Purpose: Once `/handoff` exists as a command, the next blocker is making it mean one exact thread. The current bridge can list and read sessions, and the hosted side can validate bridge bootstrap identity, but nothing today binds a Codex slash-command invocation to a reusable handoff record or returns a structured thread-bound result.

Output: A shared handoff result schema, a hosted create-or-reuse handoff route, a local `handoff codex-handoff` helper that requires thread identity and reuses `handoff launch`, and tests for reuse-first behavior plus missing-context failures.
</objective>
