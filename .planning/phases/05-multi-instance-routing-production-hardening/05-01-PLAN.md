<objective>
Build the durable relay ownership foundation for Phase 5.

Purpose: The relay cannot scale beyond one process until bridge ownership and attached-session state move into durable control-plane storage. This plan introduces the authoritative lease model, wires bridge registration and disconnects into that lease, and makes browser-facing relay code resolve ownership from durable state before trusting its local process.

Output: A durable `relay_bridge_leases` table and repository layer, a relay-instance identity helper, bridge registration/disconnect lease updates, and owner-resolution guards for browser attach/list/control paths.
</objective>
