<objective>
Add Fly-aware owner routing for browser traffic in Phase 5.

Purpose: Once durable ownership exists, the next gap is getting wrong-instance browser requests to the owning relay Machine without teaching topology to the browser. This plan implements same-app Fly replay at the relay boundary, handles replay failure safely, and validates the behavior against bridge reconnect churn.

Output: A replay helper using Fly's replay JSON format, wrong-instance replay behavior for browser WebSocket and HTTP control routes, explicit replay-failure handling, and coverage for browser replay plus bridge reconnect identity.
</objective>
