<objective>
Add the append-only audit trail and approval-state persistence required by Phase 4.

Purpose: Once durable device-session state exists, the next safety gap is the missing durable event trail. This plan centralizes load-bearing audit event names, records approval/revoke/reconnect/disconnect/ws-ticket events from the actual trust boundaries that observe them, and surfaces a compact history in the mobile device-management UI.

Output: Shared audit event constants, web + relay audit write helpers, durable audit capture for approval and trust-boundary events, and a phone-readable recent activity feed.
</objective>
