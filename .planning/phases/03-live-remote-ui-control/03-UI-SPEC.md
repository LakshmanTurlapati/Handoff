# Phase 03 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-phase for Phase 03 and intended for verification before planning.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none |
| Icon library | inline SVG or minimal custom glyphs only |
| Font | system sans stack (`system-ui`, `Segoe UI`, `sans-serif`) |

Manual design system for this phase:
- Use a lightweight product-owned component set inside `apps/web`; do not initialize shadcn for this phase while the workspace install path is blocked.
- Prefer label-first controls. Icons are supportive only and must never be the only cue for a critical action.

---

## Visual Hierarchy

- **Primary focal point:** the active live turn card.
- **Secondary focal point:** the sticky bottom composer row with `Send Prompt`, `Steer`, and `Interrupt`.
- **Tertiary focal points:** the approval card and reconnect state banner when present.
- The active live turn owns the strongest contrast and most generous vertical space.
- Older turns remain compressed but readable with lighter borders and reduced body preview.
- Icon-only actions are not allowed for primary controls, destructive controls, or approval actions. Every such control must show a text label.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline separators, status dots, icon-to-label gaps |
| sm | 8px | Tight chip spacing, compact label groups |
| md | 16px | Default card padding and element spacing |
| lg | 24px | Section padding, card-to-card gaps |
| xl | 32px | Major timeline group spacing |
| 2xl | 48px | Bottom safe-area breathing room, modal/sheet padding |
| 3xl | 64px | Major page-level spacing on large mobile screens |

Exceptions: none

Touch target rule:
- All tappable controls in the sticky composer row and approval card must resolve to a minimum 44px height even if internal spacing tokens are smaller.

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.35 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.1 |

Type rules:
- Use exactly these 4 sizes across the phase UI.
- Use exactly these 2 weights: regular `400` and semibold `600`.
- State-first turn headers use `Label`.
- Long assistant output uses `Body`.
- Session titles and major section headings use `Heading`.
- Empty states and one-screen setup states may use `Display` once per screen.

---

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F6F3ED` | Page background, main shell, large neutral surfaces |
| Secondary (30%) | `#E4DED4` | Cards, compressed turn shells, composer rail, subtle banners |
| Accent (10%) | `#0F766E` | Active live-turn border, primary CTA fill, `Jump to live`, focus ring, approval action emphasis |
| Destructive | `#B93815` | `Interrupt`, destructive confirmations, hard error emphasis only |

Accent reserved for:
- active live-turn highlight
- primary `Send Prompt` action
- `Jump to live`
- approval primary action
- focused input ring

Never use accent for:
- all links
- all interactive elements
- passive metadata
- compressed historical turns

Color behavior rules:
- Approval cards use the accent family plus strong contrast text.
- Failures use destructive color plus expanded error treatment.
- Reconnect state uses secondary surfaces with accent text, not destructive red.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Send Prompt |
| Empty state heading | No remote session yet |
| Empty state body | Open Codex on your laptop, connect the bridge, then choose a session to continue here. |
| Error state | Live connection lost. Reconnecting now. If this does not recover, reopen the session from your paired device. |
| Destructive confirmation | Interrupt Session: Stop the current Codex turn on your connected laptop? |

Additional copy rules:
- `Steer` remains the literal secondary action label in the composer row.
- `Interrupt` remains the literal destructive action label; never shorten it to `Stop`.
- Approval headers must lead with the current state, e.g. `Waiting for approval`.
- Reconnect separators use the exact label `Reconnected`.
- Avoid terminal jargon like `stdout`, `stderr`, or `SIGINT` in user-facing UI.

---

## Activity Rendering Contract

- The timeline is turn-grouped, not a raw chronological event log.
- A collapsed turn shows assistant text plus the latest live status.
- Completed tool details auto-fold; the current live step remains open.
- Older turns compress to header plus first lines while a live turn is active.
- Activity types must use distinct card treatments:
  - assistant message card
  - tool activity card
  - command execution card
  - approval card
  - system/reconnect card
  - error card
- Turn headers always lead with current state first, then the specific detail.
- The active turn must visually dominate the viewport over historical turns.

---

## Motion and Live-State Contract

- Stream assistant output in readable chunks; do not animate token-by-token jitter.
- Auto-follow remains enabled until the user scrolls away, then show `Jump to live`.
- Short reconnects keep the timeline visible and show a subtle inline banner.
- When reconnect succeeds, backfill missed events and insert a subtle `Reconnected` separator.
- After `Interrupt` is tapped, keep the timeline live and switch the control into a pending-stop state.
- Motion should be restrained: fade/slide transitions only for expand-collapse, reconnect insertion, and pending-stop feedback.
